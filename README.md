# db2git
Сохранение метаданных базы данных PostgreSql в Git.

Данный функционал периодически сохраняет объекты базы данных в файлы через pg_dump с последующей фиксацией изменений в этих файлах, с помощью системы контроля версий Git.

Базовый состав проекта:
1. gitschema.sql - SQL скрипт, создающий схему git с таблицами и функцией. Таблицы являются настроечными - позволяют указать, какие объекты будут выгружены. Функция также используется для выгрузки требуемых объектов;
2. sql2git.demo_db.sh - Bash скрипт, с помощью которого выгружаются объекты БД в каталоги;
3. Задание crontab, которое запускает скрипт sql2git.demo_db.sh;
4. Задания crontab, которые удаляют файлы, удаленных объектов БД (таблиц и функций).

Запуск демо-версии

1. Из каталога с проектом перейдите в каталог с демо
```
cd sql2git-demo
```
2. Скопируйте актуальные файлы в каталог sql2git-demo
```
cp ../gitschema.sql ../sql2git.run.sh ../sql2git.del.sh .
```
3. Запустите docker compose
```
sudo docker compose up --build
```
4. Загрузите демо-базу в контейнер
```
sudo docker exec -it -u postgres sql2git-demo-pg psql -f demo-schema.sql -d sql2git_demo_db
```
5. Загрузите конфигурационные таблицы в контейнер
```
sudo docker exec -it -u postgres sql2git-demo-pg psql -f gitschema.sql -d sql2git_demo_db
```
